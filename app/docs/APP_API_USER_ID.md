# Идентификация пользователя при вызовах App API

## Заголовок

- **Имя:** `X-User-Id`
- **Формат:** строка (UTF-8)
- **Обязательность:** необязательный; если не передан или пустой, используется `"anonymous"`.

## Откуда берётся значение на фронте

1. **Авторизованный пользователь (Nest):** после логина фронт сохраняет в `localStorage` ключ `neurochtec_user_id` через `setStoredUserId()`. В него кладётся **email** пользователя (или `dev-user` / `NEXT_PUBLIC_DEV_USER_ID` при логине в dev).
2. **При запросе к App API** используется `getStoredUserId() ?? process.env.NEXT_PUBLIC_DEV_USER_ID ?? 'anonymous'`.

Итого в заголовке уходит одна из строк:

- email пользователя (после логина в Nest),
- `NEXT_PUBLIC_DEV_USER_ID` (если задан и пользователь не логинился),
- `'anonymous'` (если ничего не задано).

## Как использует App (backend)

- Эндпоинт **POST /api/books/upload** принимает заголовок `X-User-Id` и сохраняет загруженный файл в каталог вида:
  - `storage/books/{user_id}/{book_id}/original/...`
- Так книги разделяются по пользователям; для `anonymous` все такие загрузки попадают в `storage/books/anonymous/...`.

## Совместимость

- Nest отдаёт пользователя с полями `id` (cuid) и `email`. Для App API фронт передаёт в `X-User-Id` **email** (или fallback), чтобы не привязывать App к формату id Nest. При необходимости можно позже заменить на `id` и поменять только фронт и структуру каталогов в App.
