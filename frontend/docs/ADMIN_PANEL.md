# Админ-панель НейроЧтец

## Доступ

- **URL:** `/app/admin` (после входа в кабинет)
- **Права:** только пользователи с ролью `admin`
- При отсутствии прав происходит редирект на `/app/projects`
- В боковом меню кабинета есть пункт «Админ-панель»

---

## Учётная запись администратора

Создаётся при первом запуске seed:

```
cd apps/api
npx prisma db seed
```

- **Email:** `admin@neurochtec.local`
- **Пароль:** `admin12345`

> Чтобы сделать админом существующего пользователя, в БД в таблице `User` у нужной записи нужно установить `role = 'admin'`.

## Страницы панели

### 1. Обзор (`/app/admin`)

- **Метрики:** всего пользователей, проектов, аудио; новые пользователи за 7 и 30 дней; проекты и аудио за 7 дней
- **Разбивка по статусам:** проекты, аудио, подписки
- Ссылка «Пользователи» ведёт к списку пользователей

### 2. Пользователи (`/app/admin/users`)

Таблица: email, роль, подписка, число проектов, число аудио, дата регистрации.  
Ссылка «Подробнее» открывает карточку пользователя.

### 3. Карточка пользователя (`/app/admin/users/[id]`)

- **Данные:** роль, подписка, даты создания/обновления, общее число аудио
- **Сброс пароля:** форма «Новый пароль» + «Подтверждение», кнопка «Сохранить новый пароль» (минимум 8 символов)
- **Проекты:** таблица (название, статус, язык, число аудио, дата создания)

## API (для админов)

Базовый путь: `/api/admin`. Все запросы требуют заголовок `Authorization: Bearer <access_token>` и роль `admin`.

| Метод | Путь | Описание |
|-------|------|----------|
| GET   | `/api/admin/users` | Список пользователей с счётчиками проектов и аудио |
| GET   | `/api/admin/users/:id` | Детали пользователя, проекты, подписка |
| PATCH | `/api/admin/users/:id/password` | Смена пароля. Тело: `{ "newPassword": "строка от 8 символов" }` |
| GET   | `/api/admin/metrics` | Агрегированные метрики сервиса |

---

## Метрики (`GET /api/admin/metrics`)

Возвращает объект:

- `totalUsers` — всего пользователей
- `totalProjects` — проектов (без удалённых)
- `totalAudios` — аудиофайлов
- `projectsByStatus` — `{ draft: N, ready: N, ... }`
- `audiosByStatus` — `{ queued: N, ready: N, ... }`
- `subscriptionByStatus` — `{ free: N, active: N, ... }`
- `newUsersLast7Days`, `newUsersLast30Days`
- `projectsCreatedLast7Days`, `audiosCreatedLast7Days`

## Защита на бэкенде

- **AccessAuthGuard** — проверка JWT
- **AdminGuard** — проверка `req.user.role === 'admin'` (ответ 403 при несовпадении)

Файлы: `apps/api/src/modules/auth/guards.ts`, `apps/api/src/modules/admin/admin.controller.ts`.

---

## Защита на фронтенде

- **AdminGuard** (клиент): запрос `/api/users/me`; при `role !== 'admin'` — редирект на `/app/projects`, при 401 — на `/auth/login`
- Файл: `apps/web/app/app/admin/AdminGuard.tsx`
- Подключён в `apps/web/app/app/admin/layout.tsx`

---

## Смена пароля пользователя

1. Админ-панель → Пользователи → «Подробнее» у нужного пользователя
2. Блок «Сбросить пароль»: ввести новый пароль и подтверждение (≥ 8 символов)
3. «Сохранить новый пароль»
4. После успешного сохранения пользователь может войти с новым паролем
